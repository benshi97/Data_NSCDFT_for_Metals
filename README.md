## Introduction

This repository accompanies the paper: **Practical and accurate density functionals for transition-metal heterogeneous catalysis** by Benjamin X. Shi and Timothy C. Berkelbach.

The data for plotting all of the graphs in the main text and supplemental material can be found in the folder `Data`. The notebook `analyse.ipynb` can be explored interactively with [Colab](https://colab.research.google.com/github/benshi97/Data_NSCDFT_for_Metals/blob/main/analyse.ipynb). The figures and tables generated by `analyse.ipynb` are stored in the `Figures` and `Tables` folder.

In `Scripts`, we provide the bash and [QuAcc](https://github.com/Quantum-Accelerators/quacc) workflows for running both hBEEF-vdW@BEEF-vdW as well as dhBEEF-vdW@BEEF-vdW. These can be easily modified to run other types of NSC-DFAs. Installation instructions are provided below with examples given in the `Examples` folder.

## Installation

`quacc` requires **Python ≥ 3.11** and can be installed via `pip`.  
We recommend creating a fresh Python environment using [Miniconda](https://docs.anaconda.com/miniconda/):

```bash
conda create -n quacc python=3.11
conda activate quacc
pip install "quacc @ git+https://github.com/quantum-accelerators/quacc.git"
```

### Add Project Scripts to `PYTHONPATH`

After installing `quacc`, clone this repository and add the provided scripts to your `PYTHONPATH`:

```bash
git clone https://github.com/benshi97/Data_NSCDFT_for_Metals.git
cd Data_NSCDFT_for_Metals/Scripts
export PYTHONPATH="$PWD:$PYTHONPATH"
```

> **Note:** You may wish to add the `export` line to your `.bashrc` or `.zshrc` to make the change permanent.

## VASP Environment Configuration

Before running `quacc` with VASP, configure the required environment variables:

```bash
# Parallel execution command (adjust if using srun, mpiexec, etc.)
export QUACC_VASP_PARALLEL_CMD="mpirun"

# Path to your VASP pseudopotential directory
export QUACC_VASP_PP_PATH="/path/to/vasp/pseudopotentials"

# Path to the VASP vdW kernel file
export ASE_VASP_VDW="/path/to/vasp/pseudopotentials/vdw_kernel.bindat"

# Add the VASP executable directory to your PATH
export PATH="/path/to/vasp/bin:$PATH"
```

Replace `/path/to/...` with the actual installation paths on your system.


## Paper abstract

Density functional theory (DFT) underpins modern atomistic models of heterogeneous catalysis on transition-metal surfaces.
It has opened new paradigms in rational catalyst design, predicting fundamental properties, like the adsorption energy and reaction barriers, linked to catalytic performance.
However, such applications depend sensitively on the predictive accuracy of DFT.
Achieving experimental-level reliability, so-called transition-metal chemical accuracy (13 kJ/mol), remains challenging for present density functional approximations (DFAs) and even for methods beyond DFT.
We introduce a new framework for designing DFAs tailored for molecules on transition-metal surfaces, building on recent developments in non-self-consistent DFAs.
We develop two functionals within this framework, demonstrating that transition-metal chemical accuracy can be achieved across 39 diverse adsorption reactions while delivering balanced performance for 17 barrier heights.
Moreover, the functionals correct qualitative failures of current self-consistent DFAs, including the predicted adsorption configurations of CO on Pt(111) and graphene on Ni(111).
These non-self-consistent DFAs are computationally efficient, readily integrated into existing DFT codes, and accompanied by scripts and workflows to facilitate adoption.
More broadly, this framework establishes a systematic route to developing even more accurate DFAs for heterogeneous catalysis and other complex materials systems.
