# Data_NSCDFT_for_Metals

## Introduction

This repository accompanies the paper:

**Practical and accurate density functionals for transition-metal heterogeneous catalysis**  
*Benjamin X. Shi and Timothy C. Berkelbach*

The data used to generate all figures and tables in the main text and supplementary material are stored in the `Data/` directory.

The notebook `analyse.ipynb` reproduces the analysis and figures from the paper. It can be explored interactively using Google Colab:

https://colab.research.google.com/github/benshi97/Data_NSCDFT_for_Metals/blob/main/analyse.ipynb

Outputs generated by the notebook are saved in the `Figures/` and `Tables/` directories.

The `Scripts/` directory provides both:

- **Python workflows using [quacc](https://github.com/Quantum-Accelerators/quacc)**  
- **A standalone Bash workflow**

for running the following non-self-consistent density functional calculations:

- **hBEEF-vdW@BEEF-vdW**
- **dhBEEF-vdW@BEEF-vdW**

These workflows can be adapted for other NSC-DFA calculations. Example input files are provided in the `Examples/` directory.

# Installation

`quacc` requires **Python ≥ 3.11** and can be installed via `pip`.  
We recommend using a fresh environment created with **Miniconda**.

```bash
conda create -n quacc python=3.11
conda activate quacc
pip install "quacc @ git+https://github.com/quantum-accelerators/quacc.git"
```

# VASP Environment Configuration

Before running the workflows, configure the required environment variables:

```bash
export QUACC_VASP_PARALLEL_CMD="mpirun"

export QUACC_VASP_PP_PATH="/path/to/vasp/pseudopotentials"

export ASE_VASP_VDW="/path/to/vasp/pseudopotentials/vdw_kernel.bindat"

export PATH="/path/to/vasp/bin:$PATH"
```

Replace `/path/to/...` with the appropriate paths for your system.

# Running the Python Workflow

Below is a minimal Python example demonstrating how to run the workflow.

```python
from ase import io
from nsc_dft import dhbeefvdw_flow  # or hbeefvdw_flow

atoms = io.read("POSCAR")

results = dhbeefvdw_flow(
    atoms,
    kpts=[12, 12, 1],
    calc_dir="calc_run",
)

print("dhBEEF-vdW energy:", results["dhbeef_vdw"])
print("RPA energy:", results["rpa"])
```

Run the script with:

```bash
python run_calculation.py
```

### Requirements

- `POSCAR` must be present in the working directory.
- VASP environment variables must be configured.
- The chosen `kpts` mesh should be appropriate for your system.

The workflow is **restartable**. If a calculation stops (e.g., due to walltime limits), running the script again will continue from the last completed step.

Once all calculations are finished, running the script again will simply perform the final energy analysis.

---

# Returned Energies

Both workflows return a dictionary (`results`) of total energies (in eV).

### Hybrid BEEF components

- `beef_xc_vdw` — self-consistent BEEF-vdW energy  
- `beef_xc` — BEEF exchange–correlation energy (no vdW)
- `beef_x` — BEEF exchange-only energy
- `exx` — exact exchange energy
- `hbeef_vdw` — final hBEEF-vdW energy

### Additional terms in `dhbeefvdw_flow`

- `pbe` — PBE reference energy for RPA
- `pbe_exx` — exact exchange using PBE wavefunctions
- `rpac` — RPA correlation energy
- `rpa` — total RPA energy (`pbe_exx + rpac`)
- `dhbeef_vdw` — final double-hybrid BEEF-vdW energy

All energies correspond to the same input structure.

# Bash Workflow

A standalone Bash workflow (`Scripts/nsc_dft.sh`) is also provided.  
It runs the same hBEEF-vdW and dhBEEF-vdW calculations using standard VASP input files.

The workflow runs each step in a separate directory and performs the final energy analysis automatically.

---

## Required Input Files

All VASP input files must be placed in an `inputs/` directory:

```
inputs/
├── POSCAR
├── POTCAR_DFT
├── POTCAR_RPA
├── vdw_kernel.bindat
├── KPOINTS_DFT
├── KPOINTS_RPA
├── INCAR_01_beefxc_vdw
├── INCAR_02_beefxc
├── INCAR_03_beefx
├── INCAR_04_exx
├── INCAR_05_pbe_rpa
├── INCAR_06_pbe_exx
├── INCAR_07_bands
└── INCAR_08_rpa
```

- `INCAR_*` files correspond to each step of the workflow.
- `KPOINTS_DFT` and `KPOINTS_RPA` define k-point meshes for the DFT and RPA stages.
- `POTCAR_DFT` and `POTCAR_RPA` contain the pseudopotentials.
- `vdw_kernel.bindat` is required for vdW calculations.

Example inputs are provided in `Examples/inputs/`.

---

## Usage

```bash
bash nsc_dft.sh "<MPI launcher>" <mode>
```

Where:

- `<MPI launcher>` is the command used to run VASP (`srun`, `mpirun`, etc.)
- `<mode>` must be either:

| Mode | Description |
|-----|-----|
| `hbeefvdw` | run the hBEEF-vdW workflow |
| `dhbeefvdw` | run the dhBEEF-vdW workflow (includes RPA calculations) |

### Example (SLURM)

```bash
bash nsc_dft.sh "srun --distribution=block:block --hint=nomultithread" dhbeefvdw
```

### Example (OpenMPI)

```bash
bash nsc_dft.sh "mpirun -x PATH -x LD_LIBRARY_PATH -np 16" hbeefvdw
```

---

# Features

- Each calculation step runs in its own directory
- Output files are automatically gzip-compressed
- The workflow is **restartable**
- Completed steps are automatically skipped
